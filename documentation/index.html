Introducing

Vido PHP Script provides a free and straightforward solution for hosting short videos. Enjoy seamless uploads with high-quality streaming capabilities. Share videos effortlessly and reach your audience without any limitations. This guide covers the essential steps to install and manage the website, offering a smooth experience for all users.

System Requirements

  • PHP Version: Minimum 8.1 or higher
  • Required PHP Extensions: JSON
  • Required PHP Extension: Fileinfo
  • SSL (Secure Socket Layer) enabled for secure communication
  • Folder Permissions: Ensure /upload folder has 0777 permissions
  • allow_url_fopen: Enabled
  • file_get_contents: Enabled for remote file access

Installing

  1. Open yourdomain.com/install, follow the system requirements check, and proceed to the next step.
  2. Configure your website settings, including the site name, domain, email for report notifications, upload limit, and custom text displayed to users when the upload limit is exceeded. Proceed to the next step.
  3. Set up a PIN credential to access the admin page. The password is securely hashed using the latest Argon2 algorithm to prevent unauthorized access.
  4. Completion: Ensure to delete the install folder before using the website.

Admin Access

To access the admin panel, navigate to yourdomain.com/vd-admin.php. The default PIN is 1234 or the PIN you created during setup. To change the PIN, visit the admin navigation menu.

Admin Features

  • Dashboard: Monitor current activities, including total videos, popular videos, the 5 most recent videos, and statistical graphs.
  • Manage: Manage videos directly, display video IDs for direct links, and delete videos if necessary. Sort videos by most recent or oldest.
  • SEO: All SEO elements have been implemented for optimal performance.
  • ADS: Apply any type of advertisement to customizable locations on your site.

Q&A

Q: Can this script run on shared hosting?

A: Yes, as long as the system requirements are met.

Q: Can I install this on a subdomain?

A: No, we recommend installing it on the main domain.
